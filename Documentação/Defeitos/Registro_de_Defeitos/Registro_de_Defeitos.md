# Registro de Defeitos

## Projeto

**Automação de Testes Web com Cypress**

---

## Objetivo

Este documento tem como finalidade registrar os defeitos identificados durante a execução dos testes automatizados e exploratórios da aplicação ParaBank.

Cada defeito deverá possuir um identificador único e conter informações suficientes para permitir sua reprodução e análise.

---

# Status Atual

Até o momento, **nenhum defeito foi registrado**.

Os defeitos serão adicionados conforme forem identificados durante o desenvolvimento das próximas sprints.

# Fluxo de Registro

Sempre que um defeito for identificado, deverão ser registrados:

* Identificador único (BUG001, BUG002...)
* Funcionalidade
* Título
* Descrição
* Passos para reprodução
* Resultado esperado
* Resultado obtido
* Severidade
* Prioridade
* Status
* Evidências
* Data da identificação
